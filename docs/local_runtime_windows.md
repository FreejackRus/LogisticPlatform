# Быстрый локальный runtime на Windows

Обычный Docker bind mount на Windows заметно тормозит Laravel: каждый запрос читает много PHP-файлов через файловую прослойку Windows -> Linux. Для локальной проверки интерфейса удобнее запускать проект из Docker volume.

## Запуск

```powershell
npm run dev:runtime
```

Скрипт:

- синхронизирует исходники в Docker volume `freight-exchange-runtime`;
- ставит Composer-зависимости внутри Linux volume;
- запускает контейнер `freight-exchange-app`;
- выполняет миграции;
- автоматически добавляет демо-данные, если в runtime нет грузов или машин;
- кеширует Laravel config/routes/views;
- открывает приложение на `http://127.0.0.1:8010`.

## Полный свежий запуск

```powershell
npm run dev:runtime:fresh
```

Этот вариант очищает runtime volume, пересобирает frontend assets и заново запускает миграции с демо-данными.

## Когда использовать

- страницы на `127.0.0.1:8010` грузятся слишком долго;
- после изменения PHP/React-кода нужно быстро проверить собранную версию;
- текущий контейнер начал отдавать старые ассеты или не видит новые env-настройки.

## Важно

Runtime volume является копией проекта. После изменения исходников нужно снова выполнить `npm run dev:runtime` или `npm run dev:runtime:fresh`, чтобы синхронизировать контейнер.
